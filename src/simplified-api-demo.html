🎯 简化后的 useCity API

✨ 简化内容

移除了复杂的函数重载，只保留一种简洁的调用方式：

  • ❌ 移除了传统参数调用方式
  • ❌ 移除了复杂的函数重载
  • ✅ 只保留 useCity(cityData, options) 方式
  • ✅ 所有城市属性都在 CityData 中定义

📝 新的API签名

// 简化的函数签名 export function useCity( cityData: CityData, options: UseCityOptions = {} ): CityObject // 简化的选项接口 interface UseCityOptions { radius?: number onHover?: (city: CityObject, event: MouseEvent) => void onLeave?: (city: CityObject, event: MouseEvent) => void }

🔄 对比：简化前后

❌ 简化前（复杂）

// 函数重载 export function useCity(cityData: CityData, options?: ...): CityObject export function useCity(lngLat: [number, number], name: string, options?: ...): CityObject // 复杂的实现 export function useCity( lngLatOrCityData: [number, number] | CityData, nameOrOptions?: string | ..., options?: UseCityOptions ): CityObject { // 复杂的类型判断逻辑 if (Array.isArray(lngLatOrCityData)) { // 传统方式处理 } else { // 新方式处理 } }

✅ 简化后（简洁）

// 单一函数签名 export function useCity( cityData: CityData, options: UseCityOptions = {} ): CityObject { // 直接使用 cityData 属性 const lngLat = [cityData.lng, cityData.lat] const name = cityData.name const color = cityData.color const labelOffset = cityData.labelOffset || { x: 0, y: 0, z: 0 } }

💻 使用示例

// 定义城市数据 const cityData: CityData = { name: '深圳', lng: 114.0579, lat: 22.5431, color: 0xffffff, labelOffset: { x: -5, y: 0, z: 0 } } // 创建城市对象 const city = useCity(cityData, { radius: 100, onHover: (city) => console.log('悬停城市:', city.name), onLeave: (city) => console.log('离开城市:', city.name) })

🎨 功能特性

  • 虚线连接 - 城市点与标签之间的虚线
  • 标签偏移 - 通过 CityData.labelOffset 控制
  • 颜色支持 - 通过 CityData.color 设置
  • 交互回调 - onHover 和 onLeave 事件
  • 类型安全 - 完整的 TypeScript 支持

🚀 优势

  • ✅ 更简洁 - 单一函数签名，易于理解
  • ✅ 更一致 - 所有城市属性统一管理
  • ✅ 更易维护 - 减少了复杂的类型判断
  • ✅ 更易扩展 - 新属性只需在 CityData 中添加
  • ✅ 更易测试 - 简化的逻辑更容易测试
